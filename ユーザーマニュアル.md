# 心得評価システム ユーザーマニュアル

このドキュメントは、**心得評価**（チーム内の相互評価）Webアプリを利用するときの操作説明です。一般利用者向けと管理者向けの両方を記載しています。

---

## 1. このシステムでできること

- チームに分かれたメンバー同士が、あらかじめ決めた**評価項目**ごとに **1～10点**で評価します（1が低く、10が高い目安です）。
- **自己評価**も含め、同じチームのメンバー全員が採点の対象になります。
- 評価の入力は、管理者が設定した**評価期間**のあいだだけ可能です。期間外は採点画面に進めません。
- 入力した内容の確認、チーム内の集計、月ごとの推移の閲覧ができます。

---

## 2. はじめに（アクセス・ログイン）

### アクセス先のURL

利用するURLは、組織の管理者または運用担当から案内されたものを開いてください（例: 社内ポータルやメールに記載されたアドレス）。

### ログイン

1. ログイン画面で、案内された**ユーザー名**と**パスワード**を入力します。
2. **ログイン**を押します。
3. 管理者アカウントの場合は「管理者」画面、それ以外は「心得評価」画面が開きます。

### ログアウト

画面上部の**ログアウト**を押すと、セッションが終了します。共有PCでは必ずログアウトしてください。

### 初回ログイン後のパスワード

初期パスワードは組織の運用ルールに従います。可能であれば、早めに**設定**タブからパスワードを変更してください（後述）。

---

## 3. 一般利用者向け（心得評価）

ログイン後、画面上部にあなたの**氏名**と**チーム**が表示されます。下のタブで機能を切り替えます。

| タブ名 | 内容 |
|--------|------|
| **採点** | 評価期間中に、チームメンバーへの点数を入力・保存します。 |
| **入力結果** | あなたが保存した評価内容を表で確認します。 |
| **集計** | チーム内の集計（グラフ・表など）を確認します。 |
| **月次** | 過去の月との比較・推移を確認します。 |
| **リンク集** | 運用で登録された外部サイトへのリンクが表示されます（別タブで開きます）。 |
| **設定** | ログインパスワードの変更を行います。 |

### 3.1 採点タブ

#### 評価期間について

- 評価期間**外**のときは、「評価期間ではありません」と表示され、**入力・保存はできません**。
- 期間中は、画面上部付近に「○○度の評価入力期間です（開始日 〜 終了日まで）」のような案内が表示されることがあります。

#### 点数の付け方

- 各セルに **1～10** の整数を入力します。
- **全て保存**を押すと、入力済みのセルが一括でサーバーに保存されます。
  - 空欄のままのマスは保存されません。すべての組み合わせを揃えたい場合は、未入力がないか確認してから保存してください。

#### 前回の評価をコピーする

- 前月などに保存された評価がある場合、**前回を反映**ボタンが表示されます。
- 押すと確認ダイアログのあと、**現在入力中の内容は上書き**され、前回の点数が反映されます。

#### スマートフォンでの表示

- 画面幅が狭いときは、**対象者ごと**／**項目ごと**の表示切り替えや、プルダウンで対象者・大項目を選んでから入力する形式になります。

#### パソコン（大きい画面）での表の向き

- **縦スクロールで採点（表を反転）**ボタンで、行と列の意味を入れ替えられます（項目を縦に並べて見たいとき向け）。
- 採点タブと**入力結果**タブでは、この向き設定が**共通**です。

### 3.2 入力結果タブ

- あなたが保存した評価だけが表示されます。
- 対象となる年月は画面上の説明文に従います。まだ一度も保存していない場合は「まだ評価を入力していません」と表示されます。
- PC表示では、採点タブと同様に**表を反転**できます。

### 3.3 集計タブ・月次タブ

- **集計**: 評価項目ごとのチーム内の傾向（他者から見た平均、自己評価、チーム平均など）をグラフや表で確認できます。
- **月次**: **直近3ヶ月／6ヶ月／12ヶ月／全期間**など、表示する期間の幅を選べます。項目を選んでグラフや明細を確認します。

※画面上の数値の意味や運用上の見方は、所属組織のルールに従ってください。

### 3.4 リンク集タブ

- 別サイトへのリンク一覧です。**別タブ**で開きます。社外サイトの利用規約・ログイン情報は各サービスに従ってください。

### 3.5 設定タブ（パスワード変更）

1. **現在のパスワード**、**新しいパスワード**、**新しいパスワード（確認）**を入力します。
2. 新しいパスワードは **3文字以上** です。
3. **パスワードを変更**を押します。

変更後は、次回ログインから新しいパスワードを使用してください。

### 3.6 困ったとき

- ログインできない → ユーザー名・パスワードの誤入力、大文字小文字、キーボードの入力言語を確認してください。解決しない場合は**管理者**に連絡してください。
- 採点できない → **評価期間外**の可能性が高いです。管理者に期間を確認してください。
- チームに自分がいない／メンバーがおかしい → **管理者**にメンバー登録を依頼してください。

## 4. 管理者向け

管理者でログインすると、画面上部に「**管理者**」と表示され、次のタブが使えます。

| タブ名 | 内容 |
|--------|------|
| **メンバー** | 利用者の追加、氏名の編集、チームの割り当て、パスワードの再設定など |
| **評価項目** | 採点で使う項目の登録・編集・表示順の管理 |
| **期間** | 評価入力を許可する期間（年月・開始日・終了日・有効／無効）の管理 |
| **採点調整** | 全員分の評価データの閲覧・点数の修正、CSV出力、チーム間の採点調整 |
| **集計ログ** | 利用者が「集計」タブを開いた日時の一覧（CSVでダウンロード可能） |

### 4.1 メンバー

- 新規メンバーには**ユーザー名**・**氏名**・**チーム**（A～J または **未配属**）を設定して追加します。
- **未配属**は、どのチームにも所属しないメンバーです。未配属のメンバーは採点画面に評価対象が表示されず、採点には参加しません。後からチームへ割り当てれば参加できます。
- メンバーはチームごと（および「未配属」「管理者」）のセクションに分かれて表示されます。各メンバーの選択メニューから所属チームの変更・未配属化ができます。
- 追加直後のパスワードの扱いは運用ルールに合わせて利用者に伝えてください（開発用データでは「初期パスワード＝ユーザー名」となる構成の例があります）。
- **パスワード再設定**（鍵アイコン）で、特定ユーザーのパスワードを新しい値に更新できます。
- **管理者アカウントのパスワード再設定**も、画面下部の「**管理者**」セクションにある鍵アイコンから同じ手順で行えます（管理者としてログインしている必要があります）。

### 4.2 評価項目

- **大項目**・**中項目**（名称）や説明、**表示順**を管理します。ここで登録した項目が、全利用者の採点表に反映されます。

### 4.3 期間（評価期間）

- **年月**（例: 2026-01）と**開始日**・**終了日**を指定して評価期間を登録します。
- **有効**になっている期間のうち、今日の日付が開始日〜終了日の範囲に入っているとき、利用者は採点できます。
- 複数登録する場合は、運用に合わせて**どの期間を有効にするか**を調整してください。

### 4.4 採点調整

- 画面上部の**年月**の選択で、閲覧・操作の対象月を切り替えます。
- 一覧から個別の点数を修正できます（1～10の範囲）。
- **評価データCSV出力**／**集計結果CSV出力**: ボタン押下後に表示されるダイアログで**エクスポートする年月**（例: `2026-01`）を指定してダウンロードします。
- **チーム間調整**: 項目ごとに全チーム平均と各チーム平均の差に基づき、他者スコアを一括で補正する処理です。**実行前に確認ダイアログ**が出ます。運用方針を決めたうえで使用してください。
- 画面上部の注意書きのとおり、**表示・CSV・チーム間調整は選択した年月のデータが対象**です。データがない年月を選ぶと結果が空になります。

### 4.5 集計ログ

- 一般利用者が**集計**タブを表示した日時が記録されます。
- **2か月を超えたログは自動削除**されます。
- 一覧の**CSV出力**でファイルに保存できます。

### 4.6 管理者としての注意

- 管理者アカウントのパスワードは強固なものにし、不要な共有を避けてください。
- メンバー削除や項目削除は、既存の評価データへの影響があるため、運用ルールを決めてから行ってください。

### 4.7 緊急時：管理者が誰もログインできない場合のパスワード復旧

通常、管理者のパスワードは以下の方法で再設定できます。

- 現在のパスワードが分かる場合 → 「**設定**」タブで変更
- 別の管理者がログインできる場合 → 「**メンバー**」タブの「**管理者**」セクションの鍵アイコンから再設定

**どの管理者もログインできなくなった場合のみ**、データベース（Supabase）から直接パスワードを更新します。これは最後の手段です。

> 注意: この操作はデータベースを直接変更します。対象のユーザー名を必ず確認し、設定したパスワードは一時的なものとして、復旧後すぐに「設定」タブから変更してください。

**手順（Supabase）**

1. Supabase ダッシュボードにログインし、対象プロジェクトの **SQL Editor** を開きます。
2. リポジトリ内の **`supabase/emergency_admin_password_reset.sql`** を開き、内容を SQL Editor に貼り付けます。
3. **STEP 1** を実行し、管理者アカウント（`role = 'admin'`）の一覧と `username` を確認します。
4. **STEP 2** の以下2か所を書き換えます。
   - `target_username`: リセット対象の管理者ユーザー名
   - `new_password`: 新しい一時パスワード
5. **STEP 2** を実行します（`pgcrypto` を使い bcrypt 形式でパスワードを保存します）。
6. **STEP 3** を実行し、`password_is_bcrypt` が `true` になっていることを確認します。
7. アプリのログイン画面に戻り、設定した一時パスワードでログインします。
8. ログイン後、ただちに「**設定**」タブから本来のパスワードに変更します。

※ `pgcrypto` 拡張が利用できない環境向けに、平文で一時復旧する代替手順も同ファイル末尾にコメントで記載しています（復旧後の bcrypt 化が前提です）。

---

## 5. 推奨ブラウザ・環境

- 一般的な最新の **Chrome / Edge / Firefox / Safari** の利用を推奨します。
- Cookie（ログイン状態の保持）を無効にしているとログインできないことがあります。

---

## 6. 用語の整理

| 用語 | 意味 |
|------|------|
| 評価者 | 点数を付ける人（ログイン中のあなた）。 |
| 被評価者 | 点数を付けられる人（同じチームのメンバー。自分も含む）。 |
| 評価項目 | 大項目・中項目などで整理された採点の観点。 |
| 評価期間 | 採点の入力・保存が許可される日付の範囲。 |

---

## 7. 問い合わせ先

システムのURL、ログイン情報、評価の運用ルール、数値の見方などは、**貴社・貴組織の管理者**にお問い合わせください。本マニュアルは画面上の機能に基づいた操作説明であり、社内規程に優先するものではありません。
